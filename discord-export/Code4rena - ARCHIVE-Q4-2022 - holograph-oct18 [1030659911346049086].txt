[10/24/2022 02:19] acc01ade
Any standards compliant contract can be deployed cross-chain. The HolographERC20 and HolographERC721 are the first standards enforcers in the protocol. They exist in a stack that is higher that the deployed custom code, they make sure standards are followed and enforced. Underlying custom code can be written to act however it needs to. Some simple samples of custom code are shown in the `contracts/token` dir. The protocol is built to be modular. Over time, new standards will be supported/added. ERC1155 is a perfect example of next standard being added. A user/dev can deploy contracts by creating a config, which primarily requires to specify which standard/type the contract will be following.

[10/24/2022 02:24] acc01ade
The deployed holographable contract is a stack of contracts actually. Holographer, enforcer (ERC20, ERC721, etc.), PA1D (if ERC721), and custom contract (samples: SampleERC20, SampleERC721, and CxipERC721). Function calls are routed through Holographer, which then forwards calls to enforcers, which eventually forward the calls to custom contract logic. The goal of enforcers is to do all the validation, security, and standards logic. The goal of custom contracts is to build all other logic (tokenURI for ERC721 for example).

const getEnvironment = (): Environment => {
  let environment = Environment.experimental;
  const acceptableBranches: Set<string> = new Set<string>(['experimental', 'develop', 'testnet', 'mainnet']);
  const env: string = process.env.HOLOGRAPH_ENVIRONMENT || '';
  if (acceptableBranches.has(env)) {
    console.log('HOLOGRAPH_ENVIRONMENT:', env);
